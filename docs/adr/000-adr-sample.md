# [ADR-번호] 결정 제목 (예: 임베딩 파이프라인 분리 결정)

* **상태:** [제안됨 / 승인됨 / 거부됨 / 대체됨]
* **날짜:** YYYY-MM-DD
* **작성자:** [이름]

## 1. 문맥 (Context) & 문제 상황
* 현재 어떤 상황이며, 어떤 문제가 해결되어야 하는가?
* (예: DART 데이터 크롤링과 임베딩을 동시에 수행하니 속도가 너무 느리고, 에러 발생 시 데이터가 유실됨.)

## 2. 결정 사항 (Decision)
* 우리는 무엇을 하기로 했는가?
* (예: 크롤링(Collector)과 임베딩(Vectorizer) 프로세스를 완전히 분리한다. DB의 'embedding' 컬럼이 NULL인 행만 찾아 별도로 업데이트하는 방식을 채택한다.)

## 3. 근거 (Rationale)
* 왜 이런 결정을 내렸는가? (대안과의 비교)
* (예: 실시간 처리가 불필요하며, 시스템 안정성이 최우선이기 때문.)

## 4. 결과 및 영향 (Consequences)
* **장점:** 시스템 간 결합도가 낮아져 유지보수가 쉬움. 모델 변경 시 재임베딩이 간편함.
* **단점:** 데이터가 DB에 들어온 즉시 검색되지 않고 약간의 시차(Lag)가 발생함.
